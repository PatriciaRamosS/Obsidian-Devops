Mostra o **histórico de commits** do repositório.
```
git log
```
Ele exibe informações como:

- código/hash do commit;
- autor do commit;
- data;
- mensagem do commit.
------

Mostra a **situação atual do repositório**.
```
git status
```
Ele informa, por exemplo:

- quais arquivos foram modificados;
- quais arquivos ainda não estão sendo monitorados;
- quais arquivos já foram adicionados com `git add`;
- se há commits pendentes para enviar ao GitHub.

------

Adiciona todas as alterações da pasta atual para a **staging area**, ou seja, a área de preparação do Git.
```
git add .
```
Antes de fazer um commit, você precisa dizer ao Git quais arquivos quer salvar naquele commit. O `git add .` faz isso com todos os arquivos alterados.

-------

Envia os **commits do seu repositório local** para o **repositório remoto**, como o GitHub.
```
git push 
```

------
Baixa as **alterações do repositório remoto** para o seu **repositório local**.
```
git pull
```
Ou seja, se alguém alterou algo no GitHub, ou se você alterou pelo navegador, o `git pull` traz essas mudanças para o seu computador.
