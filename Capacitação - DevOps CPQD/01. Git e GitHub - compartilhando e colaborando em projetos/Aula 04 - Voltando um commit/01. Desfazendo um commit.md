Para **desfazer um commit**, depende do que você quer fazer com as alterações daquele commit.

## 1. Desfazer o commit, mas manter as alterações nos arquivos

Use:

```bash
git reset --soft HEAD~1
```

Esse comando desfaz o último commit, mas mantém os arquivos alterados na área de preparação.

Ou seja, é como se você tivesse feito o `git add`, mas ainda não tivesse feito o commit.

Fluxo:

```bash
git reset --soft HEAD~1
git status
```

Use quando você fez o commit com a mensagem errada ou esqueceu de adicionar algo.

---

## 2. Desfazer o commit e tirar os arquivos do `git add`

Use:

```bash
git reset --mixed HEAD~1
```

Ou simplesmente:

```bash
git reset HEAD~1
```

Esse comando desfaz o último commit e mantém as alterações nos arquivos, mas fora da área de preparação.

Ou seja, os arquivos continuam modificados, mas você precisa fazer `git add` de novo.

Resumo:

```text
commit desfeito
alterações mantidas
git add desfeito
```

---

## 3. Desfazer o commit e apagar as alterações

Use com cuidado:

```bash
git reset --hard HEAD~1
```

Esse comando desfaz o último commit e também apaga as alterações dos arquivos.

Resumo:

```text
commit desfeito
alterações apagadas
```

Use apenas se tiver certeza de que não quer mais aquelas alterações.

---

## 4. Desfazer um commit que já foi enviado para o GitHub

Se o commit já foi enviado com `git push`, o mais seguro é usar:

```bash
git revert <hash-do-commit>
```

Exemplo:

```bash
git revert a1b2c3d
```

Esse comando cria um novo commit que desfaz as alterações do commit anterior.

Para ver o hash do commit:

```bash
git log
```

Depois envie para o GitHub:

```bash
git push
```

---

## Resumo simples

```bash
git reset --soft HEAD~1
```

Desfaz o commit, mas mantém tudo preparado para novo commit.

```bash
git reset HEAD~1
```

Desfaz o commit e mantém as alterações nos arquivos.

```bash
git reset --hard HEAD~1
```

Desfaz o commit e apaga as alterações.

```bash
git revert <hash>
```

Desfaz um commit que já foi enviado para o GitHub de forma mais segura.

Para estudo, a frase mais importante é:

**`reset` altera o histórico local. `revert` cria um novo commit desfazendo uma alteração, sendo mais seguro quando o commit já foi enviado ao GitHub.**
