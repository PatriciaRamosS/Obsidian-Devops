**Gist** é uma ferramenta do GitHub usada para **compartilhar pequenos trechos de código, comandos, anotações ou arquivos simples** sem precisar criar um repositório completo.

Pense assim:

```text
Repositório GitHub = projeto completo
Gist = pedaço pequeno de código ou anotação
```

## Para que serve o Gist?

Você pode usar Gist para compartilhar:

```text
- Um comando Git
- Um exemplo de código
- Um erro que apareceu no terminal
- Um arquivo de configuração
- Um passo a passo curto
- Um script pequeno
- Uma anotação técnica
```

Exemplo: em vez de criar um repositório só para mostrar este código:

```js
function soma(a, b) {
  return a + b;
}
```

Você cria um **Gist** e manda o link para outra pessoa.

## Tipos de Gist

Existem dois tipos principais:

|Tipo|Significado|
|---|---|
|**Public Gist**|Qualquer pessoa pode encontrar e acessar|
|**Secret Gist**|Só acessa quem tiver o link|

Atenção: **Secret Gist não é privado de verdade**. Ele apenas não aparece em buscas públicas. Se alguém tiver o link, consegue acessar.

## Quando usar Gist?

Use Gist quando você quer compartilhar algo pequeno, por exemplo:

```text
“Olha esse erro que apareceu no meu terminal”
“Segue o comando que eu usei”
“Segue um exemplo de função”
“Segue minha configuração do Docker”
“Segue um resumo em Markdown”
```

## Quando NÃO usar Gist?

Não use Gist para:

```text
- Projetos grandes
- Códigos com senha
- Tokens
- Chaves de API
- Dados sensíveis
- Arquivos confidenciais da empresa
```

Para projeto completo, use um **repositório GitHub**.

Para informação sensível, não use Gist público nem secret.

## Como criar um Gist

1. Acesse o GitHub.
    
2. Vá em **Gist**.
    
3. Coloque uma descrição.
    
4. Dê um nome para o arquivo, por exemplo:
    

```text
comandos-git.md
```

5. Escreva o conteúdo.
    
6. Escolha:
    
    - **Create public gist**
        
    - ou **Create secret gist**
        
7. Copie o link e compartilhe.
    

## Exemplo de Gist em Markdown

Nome do arquivo:

```text
comandos-git.md
```

Conteúdo:

````md
# Comandos básicos Git

## Ver status

```bash
git status
````

## Adicionar arquivos

```bash
git add .
```

## Criar commit

```bash
git commit -m "Atualiza arquivos"
```

## Enviar para o GitHub

```bash
git push origin main
```

````

## Diferença entre GitHub, Repositório e Gist

| Item | Uso |
|---|---|
| **GitHub** | Plataforma onde ficam repositórios e Gists |
| **Repositório** | Projeto completo com histórico, branches e arquivos |
| **Gist** | Compartilhamento rápido de código ou texto |

Resumo simples:

```text
Gist é como um “post-it de código” dentro do GitHub.
````

Ele serve para compartilhar algo pequeno, rápido e organizado.
