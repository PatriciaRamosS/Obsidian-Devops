
**Quando você clona um repositório e faz um commit, essa mudança fica salva apenas no seu repositório local, ou seja, no seu computador. Ela ainda não aparece no repositório remoto do GitHub. Para enviar essa alteração para o repositório remoto, é necessário usar o comando `git push`.**

Uma versão mais simples:

**Depois de fazer um commit, a mudança existe só na sua máquina. Para ela aparecer no GitHub, você precisa fazer um `git push`.**

Exemplo do fluxo:

```bash
git clone <link-do-repositorio>
```

Baixa o repositório do GitHub para sua máquina.

```bash
git add .
```

Prepara as alterações.

```bash
git commit -m "minha alteração"
```

Salva a alteração no histórico local do Git.

```bash
git push origin main
```

Envia a alteração para o repositório remoto no GitHub.

Então:

**commit = salva localmente**  
**push = envia para o GitHub**

### ORIGIN

![[Captura de tela de 2026-06-16 16-32-38.png]]

`origin` é o **apelido do repositório remoto**.

Nesse caso, quando você rodou:

```bash
git remote
```

o Git mostrou:

```bash
origin
```

Isso significa que seu projeto local já está conectado a algum repositório remoto, provavelmente no **GitHub**.

Pense assim:

```text
origin = nome/apelido do repositório no GitHub
```

Para ver qual é o link por trás do `origin`, rode:

```bash
git remote -v
```

Vai aparecer algo parecido com:

```bash
origin  git@github.com:usuario/repositorio.git (fetch)
origin  git@github.com:usuario/repositorio.git (push)
```

ou:

```bash
origin  https://github.com/usuario/repositorio.git (fetch)
origin  https://github.com/usuario/repositorio.git (push)
```

Explicando:

- **fetch**: de onde o Git baixa atualizações.
    
- **push**: para onde o Git envia seus commits.
    
- **origin**: o nome curto usado para não precisar digitar o link inteiro do GitHub.
    

Exemplo:

```bash
git push origin main
```

Significa:

```text
envie a branch main para o repositório remoto chamado origin
```

### Comando git clone

Quando você usa:

```bash
git clone <link-do-repositorio>
```

o Git faz duas coisas principais:

1. **Baixa uma cópia do repositório remoto** para o seu computador.
    
2. **Cria automaticamente a conexão com o repositório remoto**, usando o nome `origin`.
    

Exemplo:

```bash
git clone git@github.com:usuario/projeto.git
```

Depois que clona, se você entrar na pasta do projeto e rodar:

```bash
git remote
```

vai aparecer:

```bash
origin
```

Isso significa que o repositório local já sabe de onde ele veio.

Para ver o link completo:

```bash
git remote -v
```

Vai aparecer algo parecido com:

```bash
origin  git@github.com:usuario/projeto.git (fetch)
origin  git@github.com:usuario/projeto.git (push)
```

Ou seja:

```text
origin = repositório remoto de onde o projeto foi clonado
```

Então, quando você faz:

```bash
git push origin main
```

você está dizendo:

```text
envie meus commits locais para o repositório remoto chamado origin, na branch main
```

Resumo:

```bash
git clone
```

baixa o projeto e já configura o `origin`.

```bash
git commit
```

salva alterações localmente.

```bash
git push
```

envia alterações para o GitHub.

```bash
git pull
```

baixa atualizações novas do GitHub para sua máquina.
