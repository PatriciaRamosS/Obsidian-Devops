# Documentação de Aprendizado — Git e GitHub

## Curso: DevOps

## Módulo: Compartilhando e colaborando em projetos

## Objetivo do módulo

Este módulo apresenta os conceitos iniciais de ==**Git** e **GitHub**==, ensinando como criar uma conta, instalar o Git, criar repositórios, sincronizar projetos locais com o GitHub e compartilhar código com outras pessoas.

Ao final deste módulo, o objetivo é entender como versionar projetos e colaborar de forma organizada em ambientes de desenvolvimento, prática essencial para DevOps.

---

# 1. O que é Git?

==O **Git** é um sistema de controle de versão.== Ele permite acompanhar alterações feitas em arquivos de um projeto ao longo do tempo.

Com Git, é possível:

- Salvar versões do projeto;
    
- Voltar para versões anteriores;
    
- Ver quem alterou determinado arquivo;
    
- Trabalhar em equipe sem sobrescrever o trabalho de outras pessoas;
    
- Criar ramificações do projeto para testar novas ideias.
    

## Exemplo prático

Imagine que você está criando um projeto e faz várias alterações. Sem Git, se algo der errado, pode ser difícil voltar para uma versão anterior. Com Git, cada etapa importante pode ser salva como um “ponto de restauração”.

---

# 2. O que é GitHub?

O **GitHub** é uma plataforma online usada para armazenar repositórios Git na nuvem.

Enquanto o Git funciona localmente no computador, o GitHub permite que o projeto seja salvo e compartilhado pela internet.

Com GitHub, é possível:

- Guardar projetos online;
    
- Compartilhar código com outras pessoas;
    
- Colaborar em projetos;
    
- Criar portfólio profissional;
    
- Trabalhar com issues, pull requests e documentação;
    
- Integrar projetos com ferramentas de DevOps.
    

## Diferença entre Git e GitHub

|Ferramenta|Função|
|---|---|
|Git|Controla versões do projeto localmente|
|GitHub|Hospeda repositórios Git online|

Resumo simples:

**Git é a ferramenta de versionamento. GitHub é a plataforma onde os repositórios ficam armazenados online.**

# 3. Criando uma conta no GitHub

Para usar o GitHub, é necessário criar uma conta na plataforma.

## Passos gerais

1. Acessar o site do GitHub;
    
2. Clicar em criar conta;
    
3. Informar e-mail, senha e nome de usuário;
    
4. Confirmar o cadastro pelo e-mail;
    
5. Acessar o painel inicial da conta.
    

## Observação

O nome de usuário do GitHub é importante porque ele aparece nos links dos seus repositórios e no seu perfil profissional.

Exemplo:

```text
github.com/seu-usuario
```

# 5. O que é um repositório?

Um **repositório** é o local onde ficam armazenados os arquivos de um projeto versionado com Git.

Ele pode conter:

- Código-fonte;
    
- Documentação;
    
- Imagens;
    
- Arquivos de configuração;
    
- Histórico de alterações;
    
- Branches;
    
- Commits.
    

## Tipos de repositório

|Tipo|Descrição|
|---|---|
|Público|Qualquer pessoa pode visualizar|
|Privado|Apenas pessoas autorizadas podem acessar|

# 6. Criando um repositório no GitHub

## Passos gerais

1. Entrar no GitHub;
    
2. Clicar em **New repository**;
    
3. Definir o nome do repositório;
    
4. Escolher se será público ou privado;
    
5. Opcionalmente, adicionar um README;
    
6. Clicar em **Create repository**.
    

## Exemplo de nome de repositório

```text
curso-devops-git-github
```

## Boa prática

Usar nomes claros e sem espaços. Quando necessário, separar palavras com hífen.

Exemplo:

```text
meu-primeiro-projeto
```

# 7. Instalando o Git

Para usar Git no computador, é necessário instalá-lo.

## Verificar se o Git já está instalado

No terminal, usar:

```bash
git --version
```

Se aparecer uma versão, o Git já está instalado.

Exemplo:

```bash
git version 2.43.0
```

Se aparecer erro de comando não encontrado, será necessário instalar.

---

# 8. Instalando o Git no Ubuntu

Como você está usando Ubuntu, pode instalar com:

```bash
sudo apt update
sudo apt install git -y
```

Depois, verifique:

```bash
git --version
```

---

# 9. Configurando o Git pela primeira vez

Depois de instalar o Git, é importante configurar seu nome e e-mail.

```bash
git config --global user.name "Seu Nome"
git config --global user.email "seuemail@email.com"
```

Exemplo:

```bash
git config --global user.name "Patricia Ramos"
git config --global user.email "seuemail@email.com"
```

Para verificar as configurações:

```bash
git config --list
```

Essas informações aparecem nos commits feitos por você.

# 10. O que é git init?

O comando `git init` transforma uma pasta comum em um repositório Git.

## Exemplo

Criar uma pasta:

```bash
mkdir meu-projeto
cd meu-projeto
```

Iniciar o Git:

```bash
git init
```

Depois disso, o Git começa a controlar as alterações feitas nessa pasta.

# 11. Estados dos arquivos no Git

Um arquivo no Git pode passar por alguns estados:

|Estado|Significado|
|---|---|
|Untracked|Arquivo novo, ainda não monitorado|
|Modified|Arquivo alterado|
|Staged|Arquivo preparado para commit|
|Committed|Alteração salva no histórico|

# 12. Comandos básicos do Git

## Verificar o estado do repositório

```bash
git status
```

Mostra quais arquivos foram criados, modificados ou preparados para commit.

---

## Adicionar arquivos para commit

Adicionar um arquivo específico:

```bash
git add nome-do-arquivo
```

Adicionar todos os arquivos modificados:

```bash
git add .
```

---

## Criar um commit

```bash
git commit -m "Mensagem do commit"
```

Exemplo:

```bash
git commit -m "Adiciona arquivo inicial do projeto"
```

O commit salva uma versão do projeto no histórico do Git.

---

## Ver histórico de commits

```bash
git log
```

Versão resumida:

```bash
git log --oneline
```

---

# 13. O que é git remote?

O comando `git remote` é usado para conectar o repositório local com um repositório remoto, como o GitHub.

O repositório local fica no seu computador.

O repositório remoto fica no GitHub.

## Ver repositórios remotos configurados

```bash
git remote -v
```

## Adicionar um repositório remoto

```bash
git remote add origin URL_DO_REPOSITORIO
```

Exemplo:

```bash
git remote add origin https://github.com/seu-usuario/meu-projeto.git
```

---

# 14. Conectando o repositório local com o GitHub

Fluxo básico:

```bash
git init
git add .
git commit -m "Primeiro commit"
git branch -M main
git remote add origin https://github.com/seu-usuario/nome-do-repositorio.git
git push -u origin main
```

## Explicação dos comandos

|Comando|Função|
|---|---|
|`git init`|Inicia o Git na pasta|
|`git add .`|Adiciona os arquivos para versionamento|
|`git commit -m`|Salva uma versão do projeto|
|`git branch -M main`|Define a branch principal como main|
|`git remote add origin`|Conecta ao GitHub|
|`git push -u origin main`|Envia o projeto para o GitHub|

# 15. Sincronizando repositórios

Sincronizar significa manter o projeto local e o projeto remoto atualizados.

## Enviar alterações para o GitHub

```bash
git push
```

## Baixar alterações do GitHub

```bash
git pull
```

## Clonar um repositório existente

```bash
git clone URL_DO_REPOSITORIO
```

Exemplo:

```bash
git clone https://github.com/seu-usuario/meu-projeto.git
```

# 16. Autenticação por token no GitHub

Atualmente, o GitHub não permite mais usar senha comum para algumas operações via terminal. Em vez disso, pode ser necessário usar um **token de acesso pessoal**.

## O que é um token?

Um token é uma chave de acesso gerada no GitHub para autenticar operações como `git push` e `git pull`.

Ele funciona como uma senha, mas com permissões específicas.

## Cuidados com token

- Não compartilhar com ninguém;
    
- Não colocar em arquivos públicos;
    
- Não enviar em prints;
    
- Não salvar em repositórios;
    
- Revogar o token se ele for exposto.

# 17. Chave SSH

A chave SSH é uma forma segura de conectar seu computador ao GitHub sem precisar digitar usuário e senha toda vez.

## Ideia geral

Você gera um par de chaves:

|Chave|Função|
|---|---|
|Chave privada|Fica no seu computador e não deve ser compartilhada|
|Chave pública|É adicionada ao GitHub|

## Comando para gerar chave SSH

```bash
ssh-keygen -t ed25519 -C "seuemail@email.com"
```

Depois, é necessário adicionar a chave pública no GitHub.

# 21. Desafio prático

## Objetivo

Criar um repositório no GitHub e enviar um projeto local para ele.

## Passo a passo

1. Criar uma pasta chamada `meu-primeiro-repositorio`;
    
2. Criar um arquivo `README.md`;
    
3. Escrever uma breve descrição do projeto;
    
4. Iniciar o Git na pasta;
    
5. Fazer o primeiro commit;
    
6. Criar um repositório no GitHub;
    
7. Conectar o repositório local ao GitHub;
    
8. Enviar o projeto com `git push`.
    

## Comandos

```bash
mkdir meu-primeiro-repositorio
cd meu-primeiro-repositorio
echo "# Meu primeiro repositório" > README.md

git init
git add .
git commit -m "Adiciona README inicial"
git branch -M main
git remote add origin https://github.com/seu-usuario/meu-primeiro-repositorio.git
git push -u origin main
```
